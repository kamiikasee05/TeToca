# Sesion 2026-06-21 — Migración a Bare Metal

**Fecha:** 21 Junio 2026
**Servidor:** Ubuntu Server 24.04, 192.168.18.20
**Resultado:** Stack migrado de Docker Compose a bare metal. OpenWA es el único servicio en Docker.

Migrar todo el stack de Docker Compose (Windows/WSL2) a bare metal en Ubuntu Server para eliminar la dependencia de Docker Desktop y mejorar estabilidad.

## Fixes aplicados

### 1. WhatsApp endpoint 503 → pm2 env cache

**Problema:** `POST /api/v1/whatsapp/send` retornaba 503. El scheduler no encontraba `OPENWA_API_KEY` ni `OPENWA_SESSION_ID`.

**Diagnóstico:** `pm2 restart --update-env` lee el archivo DUMP, NO el `ecosystem.config.js`. Las variables nuevas no se cargaban.

**Fix:**
```bash
pm2 delete scheduler
pm2 start ecosystem.config.js --only scheduler
pm2 save
```

**Archivo:** `/home/kamiikasee/tetoca/ecosystem.config.js`

### 2. nginx rate limiting → 503

**Problema:** nginx devolvía 503 (Service Temporarily Unavailable) en requests al admin panel.

**Fix:** `burst=5 nodelay` → `burst=30` (removido `nodelay`). El rate limiting original era muy agresivo para el uso normal del admin.

**Archivo:** `/etc/nginx/sites-available/tuahora`

### 3. OpenWA webhook payload — body anidado

**Problema:** El handler del webhook en el scheduler esperaba `body.from` y `body.body` directamente, pero OpenWA envía el payload anidado en `body.data`.

**Fix:** Handler ahora lee `req.body?.data || req.body || {}` antes de acceder a `.from` y `.body`.

```javascript
const payload = req.body?.data || req.body || {};
const from = payload.from || '';
const message = payload.body || payload.message || '';
```

**Archivo:** `scheduler/src/routes/whatsapp.js`

### 4. Docker hostname fallbacks

**Problema:** El código hacía referencia a `tetoca_openwa` como hostname de OpenWA. En el nuevo stack, OpenWA corre en Docker y el scheduler en bare metal, así que el hostname no resuelve.

**Fix:** Cambiados todos los references de hostname:
- `appointments.js`: `tetoca_openwa` → `127.0.0.1`
- `whatsapp.js`: `tetoca_openwa` → `127.0.0.1`
- Docker container OpenWA: `--add-host scheduler:172.17.0.1` para que los webhooks de OpenWA alcancen servicios en el host.

**Archivos:** `scheduler/src/routes/appointments.js`, `scheduler/src/routes/whatsapp.js`

### 5. Address sync — admin Marca tab

**Problema:** La pestaña Marca del admin guardaba `address` y `profesional` en `config.json` pero no los sincronizaba con el scheduler.

**Fix:** `admin/save-branding.php` ahora ejecuta `curl PUT http://scheduler:3000/api/v1/providers/5` con los campos `address` y `profesional`.

**Archivo:** `landing-salon/admin/save-branding.php`

### 6. OpenWA SSRF Protection

**Problema:** OpenWA rechazaba webhooks hacia IPs privadas (172.17.0.1) por protección SSRF.

**Fix:** `WEBHOOK_SSRF_PROTECT=false` en las env vars del contenedor OpenWA.

## Lecciones aprendidas

### Quotes en PowerShell + SSH
Todos los escapes `\"` dentro de double quotes de PowerShell se rompen al pasar por sshpass → SSH → bash. Comandos remotos deben usar scripts Python scp'eados al servidor y ejecutados localmente para evitar corrupción de quoting.

### pm2 env cache
`pm2 restart --update-env` usa el archivo DUMP, no el ecosystem config. Para cambios de env vars: `pm2 delete X; pm2 start ecosystem.config.js --only X; pm2 save`.

### OpenWA como único contenedor
OpenWA necesita Chromium + Puppeteer, que requiere isolation de Docker. El resto del stack corre mejor en bare metal sin la sobrecarga de Docker Desktop.
