## EA API v1 — Campos documentados

- **Appointments:** `start`, `end`, `providerId`, `serviceId`, `customerId` (objecto plano, no anidado)
- **Customers:** `firstName`, `lastName`, `phone`, `email`
- **Booking:** `POST /api/v1/appointments` acepta estos campos directamente

## Workflows — Estabilización

### WF1 — Confirmación
- Extract-data migrado de Set node a Code node (Set no maneja bien arrays de objetos planos)
- Prefijo `549` agregado automáticamente al phone

### WF2 — Recordatorio
- Cron corregido a `0 21 * * *` (18:00 ART = 21:00 UTC)
- Dedup implementado para evitar mensajes duplicados
- IF antes del Set: evita ejecutar lógica si no hay turnos

### WF3 — Cancelación
- Filter code corregido (detección de keyword en mensaje entrante)
- Webhook 500 resuelto con `respondToWebhook` en nodo final
- Credenciales validadas
- `data.from` y `data.body` normalizados del payload OpenWA
- Cancelación vía PUT (preserva registro)

### WF4 — Reagendado
- Mismas correcciones que WF3
- Webhook estable
